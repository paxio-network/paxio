# M-L10.6 — B5 CSS port hotfix (URGENT)

**Status:** RED (architect spec, awaits frontend-dev impl)

## Контекст инцидента

После M-L10.5 merge (B5 visual port) deployed paxio.network визуально не соответствует
B5 design — components Hero/Scrolls schлoпываются в plain text на cream paper background
вместо Bloomberg-style terminal UI.

**Root cause:** frontend-dev порт-ил JSX из `docs/design/paxio-b5/components/{v_hero_b5,landing_scrolls_b5,...}.jsx`,
но **не порт-ил соответствующий CSS** из `docs/design/paxio-b5/styles/`. Components используют
~50 design-system classes (`.v-frame`, `.v-stage`, `.state-strip`, `.b3-grid`, `.btcv2-*`, etc.)
которые **отсутствуют** в `apps/frontend/landing/app/globals.css`.

| File | Source design (`docs/design/paxio-b5/styles/`) | В deploy (`apps/frontend/landing/app/globals.css`) |
|---|---|---|
| `paxio.css` | 298 строк base tokens | ✅ ported |
| `hero_variants.css` | 384 строки (.v-frame, .v-stage, .state-strip, .state-text, ...) | ❌ |
| `landing_scrolls.css` | 783 строки (scroll sub-screens) | ❌ |
| `paxio_b3_page.css` | 927 строк (.b3-grid, antithesis grid) | ❌ |
| `paxio_b5_fixes.css` | 908 строк (.btcv2-*, Bitcoin Hero) | ❌ |

**Total missing:** ~3000 строк критического CSS.

## Готово когда

1. Все 4 missing CSS файла ported в `apps/frontend/landing/app/`:
   - Опция A — append в `globals.css` (single-file simplicity, ~3500 lines)
   - Опция B — split в `app/styles/{hero,scrolls,b3,b5-fixes}.css` + import в `globals.css`
   (frontend-dev выбирает; рекомендую B — easier maintenance)
2. RED drift-guard тест `apps/frontend/landing/tests/css-class-coverage.test.tsx` GREEN —
   асертит что каждый CSS selector из 4 source files присутствует в effective stylesheet
3. Visual smoke test (Playwright OR curl-based) — landing page deployed Vercel preview
   возвращает HTML с key classes applied (например, `.state-strip`, `.b3-grid` resolve to
   computed style ≠ default)
4. `pnpm --filter @paxio/landing-app build` clean (no PostCSS warnings о undefined classes)
5. Acceptance script `scripts/verify_M-L10.6.sh` PASS

## Архитектурные требования

- **Pure CSS copy** — не модифицировать rules, только порт. Если правила конфликтуют —
  `!!! SCOPE VIOLATION REQUEST !!!` для архитектурного решения
- **Theme inheritance preserved** — `html[data-theme="dark"]` overrides из paxio.css работают
  и для ported rules (CSS variables — same scope)
- **Import order** — base tokens (paxio.css) ПЕРЕД component rules (остальные 4)
- **No Tailwind class conflicts** — ported rules не overrid'ят `.flex`, `.grid` etc.
  (если конфликт — фикс в спец case)
