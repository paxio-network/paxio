**1. Что это**

  ---------------------------------------------------------------------------------------------------
  **ОДНО ПРЕДЛОЖЕНИЕ**

  Meta-Facilitator + FAP --- платёжный движок Paxio OS:

  единственная интеграция которая даёт агенту доступ ко всем платёжным протоколам

  одновременно, включая нативный Bitcoin L1 --- без необходимости знать как работает каждый из них.
  ---------------------------------------------------------------------------------------------------

В 2026 году агентный платёжный ландшафт раздроблен между семью+ конкурирующими протоколами: x402 (Coinbase/Linux Foundation), MPP (Stripe/Tempo), ACP (Virtuals Protocol), AP2 (Google), TAP (Visa ICC), Mastercard Verifiable Intent, UCP. Каждый протокол требует отдельной интеграции, имеет свой формат proof\'а, свои правила верификации и свой SDK.

Meta-Facilitator --- агрегирующий routing layer поверх всех этих протоколов. FAP (Facilitator Aggregator Protocol) --- открытый спецификационный слой который описывает как осуществляется выбор маршрута, трансляция протоколов и верификация платежей. Агент интегрируется один раз с Paxio --- и получает доступ ко всему рынку.

  ------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
                      

  Роль в Paxio OS     Payment Layer (Слой 2). Транзакционный движок OS. Все платежи между агентами проходят через Meta-Facilitator --- он верифицирует, роутит, логирует, обновляет репутацию.

  Ключевая аналогия   Kayak для агентных платежей. Kayak не конкурирует с авиалиниями --- он агрегирует их. Meta-Facilitator не конкурирует с Coinbase x402 или Stripe MPP --- он роутит через них и добавляет то чего у них нет: Bitcoin L1, Trust Layer, Protocol Translation.

  Уникальность        Bitcoin L1 через ICP threshold ECDSA (физически невозможно на других платформах). On-chain верификация proof\'а без доверия к серверу фасилитатора. Protocol Translation между несовместимыми протоколами.

  Монетизация         Бесплатно для USDC routing (acquisition). Bitcoin fees \$0.03--0.05/подпись. FX spread 0.1--0.3% при конвертации. Routing margin при переключении между фасилитаторами.

**5. FAP --- Facilitator Aggregator Protocol**

FAP --- это открытая спецификация описывающая как агрегировать несколько платёжных протоколов за единым интерфейсом. Paxio является reference implementation FAP. Спецификация будет предложена в AAIF Agentic Commerce Working Group как открытый стандарт.

  ----------------------------------------------------------------------------------------------
  **FAP --- ЕДИНЫЙ PAYMENT INTERFACE**

  // Единый интерфейс для любого протокола:

  POST /v1/pay

  {

  \"to\": \"did:paxio:base:0x1a2b\...\", // DID получателя из Registry

  \"amount\": 0.003, // сумма в USD

  \"asset\": \"auto\", // auto \| usdc \| btc \| eth \| fiat

  \"protocol\": \"auto\", // auto \| x402 \| mpp \| tap \| btc

  \"memo\": \"translation job #4821\"

  }

  // FAP Router автоматически:

  // 1. Запрашивает Registry → какие протоколы принимает получатель

  // 2. Выбирает оптимальный маршрут (цена × скорость × надёжность)

  // 3. Выполняет транзакцию через нужный адаптер

  // 4. Верифицирует on-chain или через HTTPS Outcall

  // 5. Логирует в ICP audit log

  // 6. Обновляет репутацию в Registry

  // Клиент получает:

  { \"status\": \"confirmed\", \"tx_id\": \"0xabc\...\", \"protocol_used\": \"x402-coinbase\",

  \"settled_at\": \"2026-04-14T10:23:45Z\", \"fee_paid\": 0.0001 }
  ----------------------------------------------------------------------------------------------

**FAP Routing Algorithm**

  --------------------------------------------------------------------------------
  **КАК FAP ВЫБИРАЕТ МАРШРУТ**

  1\. Получить список поддерживаемых протоколов агента-получателя из Registry

  → Agent Card.payment_accepts: \[\'x402\', \'btc-l1\', \'mpp\'\]

  2\. Для каждого поддерживаемого протокола вычислить score:

  score = (availability_weight × uptime) +

  (cost_weight × (1 / fee)) +

  (speed_weight × (1 / settlement_time_ms))

  Веса по умолчанию: availability=0.4, cost=0.35, speed=0.25

  Enterprise может задать собственные веса через API

  3\. Если запрошенный asset не совпадает с preferred asset получателя:

  → проверить доступность Protocol Translation

  → добавить translation cost к итоговому fee

  4\. Circuit breaker: исключить фасилитаторы с uptime \< 95% за последние 5 мин

  5\. Выбрать протокол с наибольшим score

  → fallback: если выбранный протокол отвалился в процессе → retry с #2
  --------------------------------------------------------------------------------

**6. Bitcoin L1 --- уникальная возможность**

Bitcoin L1 routing --- единственная возможность в Meta-Facilitator которая технически невозможна без ICP infrastructure. Все остальные протоколы можно реализовать на обычном стеке. Bitcoin L1 без кастодиана --- только через ICP threshold ECDSA.

  -----------------------------------------------------------------------
  **КАК РАБОТАЕТ BITCOIN L1 ТРАНЗАКЦИЯ**

  Агент инициирует: pay(did, 0.00005 BTC, \'btc-l1\')

  FAP Router → Bitcoin L1 Adapter → Paxio Wallet Canister (Продукт 3):

  Wallet Canister на ICP:

  ├── Проверяет Security Sidecar (Transaction Intent Verifier)

  ├── Запрашивает threshold ECDSA подпись у 13+ ICP узлов

  ├── Каждый узел держит часть приватного ключа (Shamir Secret Sharing)

  ├── Узлы совместно формируют подпись без раскрытия ключа

  └── Формирует и отправляет подписанную UTXO транзакцию на Bitcoin L1

  Verification Layer:

  ├── Ждёт 1 confirmation (≈ 10 минут)

  ├── Верифицирует через ICP Chain Fusion Bitcoin integration

  └── Обновляет статус транзакции → confirmed

  Стоимость: \$0.026 (threshold ECDSA computation)

  Наша цена: \$0.03--0.05/подпись → маржа 15--92%

  Альтернатива: нет. Ни один другой фасилитатор не поддерживает BTC L1.

**8. Standalone стратегия --- успех без полного Paxio OS**

Meta-Facilitator может и должен быть успешным как standalone продукт. Разработчик или DeFi протокол может использовать только Facilitator --- без Registry, без Wallet, без Security Layer. Ценность уже есть в одном продукте.

  ----------------------------------------------------------------------------------
  **STANDALONE VALUE PROPOSITION**

  «Один API для всех агентных платёжных протоколов + Bitcoin L1 + on-chain агенты»

  Разработчик приходит с одной проблемой:

  • «Мой агент должен принимать платежи от любого другого агента»

  • «Мой DeFi бот должен платить off-chain оракулу»

  • «Мне нужен Bitcoin L1 для агентных транзакций»

  • «Мой клиент на Stripe MPP хочет платить моему агенту на x402»

  Ни одна из этих проблем не требует весь Paxio OS.

  Facilitator решает каждую из них отдельно.
  ----------------------------------------------------------------------------------

  **КРИТИЧЕСКИЙ ВОПРОС --- ПОЧЕМУ НЕ ПРОСТО COINBASE?**

  Разработчик с одним агентом на Base: npm install x402 → работает → бесплатно → зачем Paxio?

  Ответ: если ему нужен только USDC на Base --- Coinbase действительно лучше. Мы не конкурируем.

  Paxio Facilitator нужен когда:

  → Агент должен принимать ЛЮБОЙ токен от ЛЮБОГО агента (не только USDC/Base)

  → Агент должен платить или принимать Bitcoin L1 (только мы)

  → On-chain агент должен платить off-chain агенту (только мы)

  → Агенты на разных протоколах (MPP ↔ x402) должны взаимодействовать (только мы)

  → Нужна on-chain верификация proof без доверия серверу Coinbase (только мы через Chain Fusion)

  Это 4 чётких use case где мы единственные. Один из них достаточен для standalone успеха.

**10. Что НЕ входит в Meta-Facilitator**

  --------------------------------------------- ------------------------------------------------------------------------------------------------------- -----------------------------------
  **Что НЕ делаем**                             **Почему**                                                                                              **Где это решается**

  Не генерируем кошельки агентов                Кошелёк --- отдельный продукт (Trust Layer).                                                            Продукт 3: Paxio Wallet + Adapter

  Не запускаем Security checks самостоятельно   Security проверки делегируются Security Sidecar.                                                        Продукт 4: Security Layer

  Не являемся custody для средств               Не держим пользовательские funds. Capital float --- только для Protocol Translation, не для хранения.   ICP canisters без admin доступа

  Не даём кредит или займы агентам              Только routing существующих средств. Никакого leverage.                                                 Вне scope

  Не определяем exchange rate самостоятельно    Используем market rate с минимальным spread. Rate источник --- on-chain DEX или oracle.                 Chainlink / on-chain DEX

  Не обрабатываем chargeback для crypto         Crypto транзакции необратимы. Refund --- новая транзакция, не chargeback.                               Refund endpoint (v1.2)

  Не верифицируем identity плательщика          KYA (Know Your Agent) --- функция Security и Compliance layers.                                         Продукты 4, 6

**11. Интеграции внутри Paxio OS**

  -------------------------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------ --------------------------------------------------
  **Продукт Paxio**                **Как интегрируется**                                                                                                                                        **Направление**

  Paxio Wallet (Продукт 3)         Все исходящие Bitcoin L1 транзакции идут через Wallet Canister. Facilitator вызывает Wallet для threshold ECDSA подписи. Wallet содержит Security Sidecar.   Facilitator → Wallet (BTC signing)

  Compliance Layer (Продукт 6)     Каждая транзакция логируется в ICP Audit Log canister (часть Compliance). Compliance Layer читает лог для EU AI Act reporting.                               Facilitator → Compliance (tx log)

  Intelligence Layer (Продукт 7)   Facilitator emits transaction events: protocol, volume, timing, error rates. Intelligence Layer агрегирует для market analytics и fraud models.              Facilitator → Intelligence (events)

**12. Нефункциональные требования**

  -------------------- ---------------------------------------------------------------------------------------------------------- ---------------------------------------------------------------------------------------
  **Категория**        **Требование**                                                                                             **Как достигается**

  Производительность   USDC routing: p50 \< 200ms end-to-end, p99 \< 1s. Bitcoin L1: 1--60 мин (зависит от Bitcoin block time).   Stateless Routing Engine + Redis cache для routing decisions + async processing

  Доступность          99.9% для USDC routing. Bitcoin L1 follows Bitcoin network availability.                                   Multi-region deployment + circuit breakers + automatic failover к backup фасилитатору

  Атомарность          Protocol Translation операции: atomic swap или полный rollback. Нет частичного выполнения.                 ICP canister transactions + capital float escrow + compensation logic

  Идемпотентность      Повторный вызов с тем же payment_id → возврат кешированного результата, не новая транзакция.               Idempotency key в каждом запросе, хранение в Redis с TTL=24h

  Аудитируемость       Каждая транзакция логируется on-chain на ICP. Невозможно удалить или изменить.                             ICP Audit Log canister без admin key

  Compliance           OFAC screening для каждого нового адреса. Логирование для EU AI Act.                                       Sanctions Oracle + ICP immutable log
