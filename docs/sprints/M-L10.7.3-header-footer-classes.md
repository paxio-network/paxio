# M-L10.7.3 — Header + Footer refactor: inline styles → design class names

**Status:** RED
**Owner:** frontend-dev
**Effort:** 2–3 hours
**Branch:** `feature/M-L10.7.3-header-footer-classes`

## Цель

Header.tsx (385 lines) + Footer.tsx (180 lines) построены на **16 inline `style={}` блоках вместо design class names**. CSS rules для `.hdr-*`, `.foot-*` УЖЕ port'нуты в landing's `paxio-b3-page.css` (M-L10.6), но компоненты эти rules игнорируют — рендерят inline styles.

Результат: визуально расходится с design source. Header brand block / nav links / actions / CTAs не используют design typography + spacing rules.

## Конкретные расхождения

### D-1. Header — `id="paxio-header"` + `<div class="hdr-inner">` structure

Design source (Paxio-B5.html line 26):
```html
<header id="paxio-header" data-screen-label="00 Header">
  <div class="hdr-inner">
    <a href="#registry" class="hdr-brand" aria-label="Paxio home">
      <svg class="hdr-mark" .../>
      <span class="hdr-wordmark">Paxio</span>
      <span class="hdr-tagline">Financial OS · Agentic Economy</span>
    </a>
    <nav class="hdr-links" aria-label="Primary">
      <a href="#registry">Registry</a>
      <a href="#radar">Radar</a>
      <a href="https://docs.paxio.network">Docs</a>
    </nav>
    <div class="hdr-actions">
      <button class="hdr-theme-btn" .../>
      <a href="#quickstart" class="hdr-cta hdr-cta-outline">Install SDK</a>
      <a href="https://registry.paxio.network" class="hdr-cta hdr-cta-primary">Open Registry →</a>
      <span class="hdr-live"><span class="pulse-dot"/><span>live</span></span>
    </div>
  </div>
</header>
```

Landing impl (`packages/ui/src/Header.tsx`):
- `<header className="paxio-header">` (НЕ `id="paxio-header"` — different selector!)
- inline `style={{ position: 'sticky', top: 0, zIndex: 50, background: 'rgba(...)', backdropFilter: 'blur(10px)', borderBottom: '1.5px solid var(--ink-0)' }}` — overrides CSS file rule
- inner `<div style={{ maxWidth: '1440px', margin: '0 auto', paddingInline: 'clamp(24px, 4vw, 56px)', height: '64px', display: 'grid', gridTemplateColumns: 'auto 1fr auto', alignItems: 'center', gap: '24px' }}>` — replaces `.hdr-inner` class
- `<a>` brand: inline `style={{ display: 'flex', alignItems: 'center', gap: '10px', textDecoration: 'none', color: 'inherit' }}` — replaces `.hdr-brand`
- `<span>` wordmark: inline `style={{ fontFamily: 'var(--f-display)', fontWeight: 600, fontSize: '18px', letterSpacing: '-0.01em', color: 'var(--ink-0)' }}` — replaces `.hdr-wordmark`

**Fix:** rewrite Header.tsx using design class names (`hdr-inner`, `hdr-brand`, `hdr-mark`, `hdr-wordmark`, `hdr-tagline`, `hdr-links`, `hdr-actions`, `hdr-theme-btn`, `hdr-cta`, `hdr-cta-outline`, `hdr-cta-primary`, `hdr-live`). Use `id="paxio-header"` (not className). Drop inline styles for layout properties (CSS rules in `paxio-b3-page.css` will take over).

## Архитектурные требования

- **Pure refactor** — design CSS already ported in `paxio-b3-page.css`, just remove inline styles + use class names
- **No new CSS rules** — все стили уже в `paxio-b3-page.css` (M-L10.6 ported byte-identical)
- **Tests SACRED** — frontend-dev НЕ trogan test files
- **Inline styles OK ONLY for SVG** (e.g. `<svg style={{ display: 'inline-block', verticalAlign: '-3px' }}>`) — это design-source pattern, не layout
- **CSS Coverage drift-guard (M-Q20)** — каждый new className используемый в JSX должен резолвиться в `paxio-b3-page.css`. После refactor coverage стабилизируется (классы в CSS уже есть).
