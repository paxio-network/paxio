**3.1 Bitcoin DCA Agent**

  --------------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **Параметр**                **Детали**

  Что делает                  Автономно конвертирует доходы агента в BTC по расписанию: USDC → ckBTC через ICPSwap (on-chain DEX) → BTC L1 через threshold ECDSA. Полностью без участия человека.

  Для кого в первую очередь   Агенты в Paxio Registry которые зарабатывают USDC: Translation Agent, Security Scanner, Complior Agent --- любой агент с доходом хочет держать его в BTC.

  Уникальность                Единственный DCA агент с настоящим BTC L1 без кастодиана. Swan/River требуют человека. cbBTC --- wrapped. Только мы делаем UTXO.

  Fee estimation              Canister проверяет mempool fee rate перед покупкой. Если fee \> X% от суммы --- откладывает до снижения.

  Приоритет                   🔴 MVP --- первый Revenue агент. Простая логика, очевидная ценность, существующий спрос.

**3.2 Bitcoin Escrow Agent**

  ------------------------------------------------------------------------------------------
  **ESCROW FLOW**

  Агент A (покупатель) → Escrow Canister:

  «Заблокируй 0.001 BTC для задачи #4821. Исполнитель: Agent B. Timeout: 48h»

  Escrow Canister → Агент B: «Средства заблокированы, начинай работу»

  Агент B выполняет → Registry: «задача #4821 завершена, hash: 0xabc\...»

  Escrow Canister верифицирует через Registry Validation:

  Proof принят → threshold ECDSA → 0.001 BTC → адрес Агента B → DONE

  Timeout без proof → threshold ECDSA → возврат Агенту A → DONE

  Арбитраж: третий агент-арбитр решает спор до timeout. Решение исполняется автоматически.
  ------------------------------------------------------------------------------------------

  ---------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------
  **Параметр**           **Детали**

  Для кого               Любая A2A сделка где нужно доверие без третьей стороны. Оркестратор нанимает специалиста. DAO платит контрактору. Агент покупает данные у агента.

  Почему BTC а не USDC   USDC можно заморозить Circle. BTC нельзя. Для крупных сделок между незнакомыми агентами BTC escrow = максимальное доверие.

  Monetization           0.1% от суммы escrow. При \$1K = \$1. При \$100K = \$100.

  Приоритет              🔴 MVP --- базовый примитив A2A коммерции. Прямая синергия с Registry.
  ---------------------- ---------------------------------------------------------------------------------------------------------------------------------------------------

**3.3 Bitcoin Streaming Payments**

Первый payroll примитив для агентной экономики. Inspired by Sablier (EVM streaming) --- адаптирован для Bitcoin через ICP threshold ECDSA и ckBTC.

  -----------------------------------------------------------------------------------
  **КАК РАБОТАЕТ STREAMING**

  Оркестратор нанимает Translation Agent на проект длительностью 8 часов:

  rate: 0.00000100 BTC/секунда (≈ \$0.008/sec при BTC \$80K)

  total: 0.0288 BTC (≈ \$2304 за 8 часов)

  Оркестратор → Streaming Canister: lock 0.0288 BTC + start_stream(agent_did, rate)

  Каждую секунду пока агент активен:

  ckBTC баланс Translation Agent += 0.00000100 ckBTC

  (ICP timer, 1-second tick, мгновенные ckBTC транзакции)

  Агент завершил задачу или стриминг остановлен:

  ckBTC → BTC L1 batch settlement (раз в 24 часа для экономии fees)

  Остаток → возврат оркестратору

  Агент получает оплату в реальном времени за реальную работу.

  Нет invoice. Нет ожидания. Нет доверия к оркестратору.
  -----------------------------------------------------------------------------------

  ------------------ -----------------------------------------------------------------------------------------------------------------------------------------------
  **Параметр**       **Детали**

  Для кого           Долгосрочные задачи: analysis agents, monitoring agents, research agents. Любая работа где агент активен часами.

  Почему это важно   Первый true payroll для агентной экономики. Агент не ждёт invoice approval. Оркестратор не платит авансом без гарантий. Обе стороны защищены.

  ckBTC vs BTC L1    Streaming идёт в ckBTC (мгновенно, нет fees для каждого тика). Settlement на BTC L1 batch раз в 24h. Лучший UX + минимальные fees.

  Monetization       0.2% от stream объёма. При \$2304 стриме = \$4.60.

  Приоритет          🔴 MVP --- нигде не существует. Первый в мире payroll примитив для AI агентов.

**3.4 Bitcoin Reputation Stake**

Reputation score в Registry --- цифра. Bitcoin stake --- деньги. Это принципиально разные уровни commitment. Агент который стейкает 0.01 BTC перед сделкой говорит: «я уверен в себе настолько что ставлю реальные деньги».

  --------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **Параметр**          **Детали**

  Как работает          Покупатель запрашивает stake перед крупной сделкой. Seller Agent стейкает BTC в Escrow Canister. Successful delivery → stake возвращается + reputation +. Failed delivery → stake слэшится (частично или полностью) → покупателю.

  Slash механизм        Configurable: при timeout → 100% slash. При partial delivery → proportional. При dispute → арбитр решает. Все правила в canister state --- нельзя изменить после стейкинга.

  Registry интеграция   Агенты с активным stake получают «Staked Agent» badge в Registry. Видно в поиске. Фильтрация по minimum_stake для высокорисковых задач.

  Для кого              Агенты с высоким ticket: MITRE Analysis (\$5/модель), Enterprise Compliance (\$50/скан), Bitcoin Treasury Management. Покупатели хотят гарантии для крупных платежей.

  Монетизация           0.05% от stake. При 0.1 BTC stake = \$4 при текущей цене. Плюс slash proceeds идут частично в Paxio insurance pool.

  Приоритет             🔴 MVP --- уникальный механизм. Делает reputation score не просто цифрой а деньгами.
  --------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

**3.5 Bitcoin Treasury Agent**

  ----------------------- -----------------------------------------------------------------------------------------------------------------------------------------------------------
  **Параметр**            **Детали**

  Что делает              Autonomous CFO для корпоративного BTC treasury: хранение, allocation rules, spending policies, reporting, ребалансирование при отклонении от target.

  Multi-sig архитектура   (m-of-n) через ICP threshold ECDSA. Например: 3 из 5 approvers нужны для транзакций выше \$10K. Approvers --- canisters или Internet Identity. BFT-aware.

  Spending policies       Daily/monthly лимиты по категориям. Whitelist адресов. Нет одноразовых approvals --- политики в canister state, неизменяемы без governance процедуры.

  Ребалансирование        Target allocation: например 30% BTC, 50% USDC, 20% ETH. При отклонении \>5% → auto-rebalance через ICPSwap. Configurable thresholds.

  Reporting               Daily snapshot: BTC balance, USD value, allocation %, P&L за период, топ транзакции. On-chain signed --- верифицируемо аудитором.

  Для кого                Компании с BTC на балансе. После MicroStrategy trend --- сотни компаний. DAO treasury. Агентные платформы с большим BTC float.

  Monetization            0.1% AUM/год + \$99/мес за dashboard. При \$1M AUM = \$1000/год.

  Приоритет               🟡 v1.1 --- enterprise sales цикл, большой ticket.

**3.6 Bitcoin Yield Agent**

  -------------------- --------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **Параметр**         **Детали**

  Что делает           Puts idle ckBTC to work. Конвертирует неиспользуемый ckBTC в yield-generating позиции: ICPSwap liquidity provision, lending протоколы на ICP. 2-5% APY.

  Почему это важно     Treasury держит BTC долгосрочно. Но 100% в холодном хранении = упущенная доходность. Yield Agent превращает dead asset в earning asset.

  Механизм             Treasury canister → Yield Agent canister: «используй 20% нашего ckBTC для yield». Agent предоставляет ликвидность в ICPSwap BTC/USDC пул. Fees accumulate в ckBTC.

  Risk management      Configurable: max allocation (%), max IL exposure, min liquidity ratio для withdrawal. Auto-withdrawal если conditions нарушены.

  Целевая доходность   ICPSwap BTC/USDC pool: \~2-4% APY. Lending на NNS-based protocols: \~3-5% APY. Без custodian, on-chain верифицируемо.

  Для кого             Treasury Agents с долгосрочной позицией. DAO treasury. Long-term Bitcoin holders которые не хотят продавать но хотят доходность.

  Приоритет            🟡 v1.1 --- зависит от зрелости ICP DeFi экосистемы.

**3.9 Trustless Inheritance**

  ------------------------------ -----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **Параметр**                   **Детали**

  Что делает                     Если владелец (или агент-хранитель) не активен N месяцев → BTC автоматически передаётся указанным наследникам. Time-locked. Без адвокатов. Без custodian.

  Механизм проверки активности   Canister проверяет last_activity через Internet Identity API. Владелец может сделать check-in в любой момент (продлевает таймер). При N месяцах без check-in → начинается countdown → уведомление → исполнение.

  Наследники                     До 5 адресов с долями. Условия: time-lock (минимальный возраст получателя), split rules (50/50, custom %), emergency override через multi-sig.

  Рыночный нарратив              Estate planning: \$68T рынок передачи состояния в США. Потерянный BTC = потерянный навсегда (\~4M BTC уже потеряно). Trustless Inheritance решает это математически.

  Уникальность                   Единственный trustless BTC inheritance механизм для AI агентов. Не требует юриста, не требует Hardware Security Module, не требует доверия третьей стороне.

  Приоритет                      🟢 v1.2 --- сильный press нарратив, долгосрочный продукт.

**5. Bitcoin L1 vs Lightning --- осознанный выбор**

Часто задаваемый вопрос: почему не Lightning Network? Lightning имеет смысл для определённых задач --- микроплатежи, instant settlement, high-frequency. Но для наших агентов L1 правильнее по трём причинам.

  ----------------------- ----------------------------------------------------------------------------------------------------- ---------------------------------------------------------------------------------------------------------
  **Сравнение**           **Bitcoin L1 (наш выбор)**                                                                            **Lightning Network**

  Settlement finality     Абсолютная --- после N confirmations транзакция неизменяема. Escrow и Treasury требуют L1 finality.   Conditional --- нужно держать канал открытым. При закрытии канала = on-chain.

  Суммы транзакций        Оптимален для \$10+ транзакций. DCA \$100/неделю, Treasury \$10K+, Escrow \$1K+.                      Оптимален для микроплатежей \$0.01-\$10. Не наш primary use case.

  Управление ключами      Threshold ECDSA --- один ключ, никакого channel management.                                           Channel management сложен: opening, closing, routing, liquidity management.

  ICP интеграция          Chain Fusion Bitcoin Integration --- нативная в ICP. Прямая интеграция.                               Нет нативной Lightning интеграции в ICP. Требует external LSP.

  Custodial риск          Некастодиальный по архитектуре --- threshold ECDSA.                                                   Часто используется с hosted LSP (Lightspark = кастодиальный).

  Когда Lightning лучше   ---                                                                                                   Streaming sub-satoshi micropayments. Real-time gaming. High-frequency API calls. Не наши primary cases.
  ----------------------- ----------------------------------------------------------------------------------------------------- ---------------------------------------------------------------------------------------------------------

  -----------------------------------------------------------------------
  **ckBTC --- ЛУЧШЕЕ ИЗ ОБОИХ МИРОВ**

  ckBTC на ICP = скорость Lightning + finality Bitcoin L1:

  Скорость: 2-секундные транзакции (ICP consensus)

  Стоимость: нет miner fees для ckBTC transfers

  Конвертация: ckBTC → BTC L1 в 30-60 минут (на-chain settle)

  Trust: 1:1 backed, verifiable on-chain, no custodian

  Streaming Payments используют ckBTC для per-second ticks.

  Settlement происходит на BTC L1 batch раз в 24h (минимальные fees).

  Лучший UX без компромисса по безопасности.
