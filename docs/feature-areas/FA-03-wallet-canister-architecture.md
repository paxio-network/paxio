**1. Что это**

  ----------------------------------------------------------------------------
  **ОДНО ПРЕДЛОЖЕНИЕ**

  Paxio Wallet + Adapter --- кошелёк для AI агентов

  который инжектируется в любой существующий фреймворк одной командой,

  даёт нативный Bitcoin L1 адрес без кастодиана,

  и защищает каждую транзакцию через Security Sidecar на уровне архитектуры.
  ----------------------------------------------------------------------------

Продукт состоит из двух связанных частей. Paxio Wallet --- это ICP canister который хранит ключи агента, управляет балансами и подписывает транзакции через threshold ECDSA. Wallet Adapter --- это тонкий слой интеграции который позволяет подключить Wallet к любому существующему агентному фреймворку без переписывания кода: одна строка для Python/TypeScript, четыре строки JSON для MCP, ноль строк для HTTP Proxy.

Главное техническое отличие от всех конкурентов: ключи агента никогда не существуют в одном месте. Threshold ECDSA распределяет ответственность за подпись между 13+ ICP узлами --- нет single point of failure, нет кастодиана, нет custodial risk. Это не обёртка над кастодиальным кошельком --- это первый в мире некастодиальный кошелёк с нативным Bitcoin L1 для AI агентов.

  ------------------- -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
                      

  Роль в Paxio OS     Trust Layer (Слой 3), компонент 1 из 3. Хранилище активов агента и исполнитель транзакций. Все платежи через Meta-Facilitator в конечном счёте подписываются Wallet Canister.

  MetaMask аналогия   MetaMask дал кошелёк людям в браузере. Paxio Wallet даёт кошелёк агентам в коде. Та же идея --- другой масштаб и другой actor. 3M+ агентов без кошелька сегодня.

  Два компонента      Wallet Canister (ICP) --- хранит ключи, подписывает транзакции, содержит Security Sidecar. Wallet Adapter (SDK/MCP/Proxy) --- тонкий клиент для подключения любого фреймворка к Canister.

  Уникальность        Нативный BTC L1 без кастодиана (threshold ECDSA). Security Sidecar встроен архитектурно --- bypass невозможен. Три способа подключения без переписывания кода агента.

**1.1 Кошелёк vs Адаптер --- важное разграничение**

Частый вопрос: «кошелёк --- это MCP?» Нет. MCP --- это один из трёх способов подключить кошелёк к агенту. Сам кошелёк живёт в ICP canister и там хранятся реальные ключи и деньги. Адаптер --- только интерфейс доступа к нему.

**4.3 Guard Agent --- INPUT защита перед агентом**

Security Sidecar защищает OUTPUT агента (что он делает с деньгами). Guard Agent (Продукт 4) защищает INPUT агента (что он получает из внешнего мира). Это два разных рубежа одной системы безопасности.

  -------------------------------------------------------------------------------

  (PDF/email/API) (ML, \<200ms) (reasoning) (Rust, ICP) (ICP)

  │── incoming ──────────►│ │ │ │

  │ │── CLEAN ─────────────►│ │ │

  │ │── BLOCK (quarantine) ──│ (stop) │ │

**5. Threshold ECDSA --- почему это важно**

Threshold ECDSA --- это криптографический протокол многостороннего вычисления (MPC) который позволяет группе узлов совместно создавать цифровую подпись без того чтобы любой из них когда-либо видел полный приватный ключ. ICP реализует это в production с 2021 года как core infrastructure.

  --------------------------------------------------------------------------------
  **КАК РАБОТАЕТ THRESHOLD ECDSA НА ICP**

  Обычный кошелёк (Metamask, Coinbase Wallet):

  Приватный ключ → хранится в одном месте (браузер, сервер, HSM)

  Один взлом = потеря всех средств

  Custodial risk: провайдер может заморозить или потерять ключ

  Threshold ECDSA на ICP:

  Приватный ключ НИКОГДА не существует целиком

  13+ ICP узлов (разные организации, разные юрисдикции) держат key shares

  Для подписи нужно согласие t-из-n узлов (например 9 из 13)

  Если несколько узлов взломаны --- ключ всё равно в безопасности

  ICP как организация не может получить доступ к ключу

  Что это значит для агента:

  ✓ Настоящий Bitcoin L1 адрес (bc1q\...) --- не wrapped, не custodial

  ✓ Paxio не контролирует средства агента --- даже если захочет

  ✓ Нет single point of failure --- ни технического ни юридического

  ✓ On-chain верифицируемо: публичный адрес деривируется из threshold public key
  --------------------------------------------------------------------------------

**10. Что НЕ входит в Wallet**

  ---------------------------------------- ------------------------------------------------------------------------------------ ---------------------------------------
  **Что НЕ делаем**                        **Почему**                                                                           **Где это решается**

  Не роутим платежи между протоколами      Routing --- задача Meta-Facilitator. Wallet только подписывает.                      Продукт 2: Meta-Facilitator + FAP

  Не предоставляем UI кошелька для людей   Wallet для агентов, не для людей. Dashboard --- только для управления агентом.       Обычные кошельки (Metamask, Ledger)

  Не кастодируем средства от имени Paxio   Threshold ECDSA означает что Paxio не контролирует ключи. Это не custodial сервис.   ICP threshold ECDSA (некастодиальный)

  Не поддерживаем Lightning Network        Lightning = L2, требует channel management, другая архитектура. Не наш фокус.        Lightspark (если нужен Lightning)

  Не делаем полноценный DEX                Auto-convert использует ICPSwap как внешний сервис. Не строим свой DEX.              ICPSwap, Sonic DEX на ICP

  Не проводим KYC пользователей            KYC/KYA --- функция Compliance Layer. Wallet знает только DID агента.                Продукт 6: Compliance Layer

  Не страхуем средства                     Нет custodial responsibility = нет страхования. Агент несёт свои риски.              Отдельные DeFi insurance протоколы

**11. Интеграции внутри Paxio OS**

  -------------------------------- --------------------------------------------------------------------------------------------------------------------------------------------------------------- -----------------------------------------------------
  **Продукт Paxio**                **Как интегрируется**                                                                                                                                           **Направление**

  Universal Registry (Продукт 1)   При создании кошелька: агент авто-регистрируется в Registry, BTC адрес привязывается к DID. Поле bitcoin_enabled = true, wallet_type = threshold_ecdsa.         Wallet → Registry (auto-registration)

  Meta-Facilitator (Продукт 2)     Все исходящие платежи через Facilitator проходят Wallet для подписи. Facilitator не может подписать транзакцию в обход Security Sidecar.                        Facilitator → Wallet (signing + security gate)

  Security Layer (Продукт 4)       Security Sidecar встроен в Wallet Canister --- это не отдельный вызов. Политики хранятся в canister state. Полная детальная аналитика угроз --- в Продукте 4.   Wallet ↔ Security (integrated + extended analytics)

  Compliance Layer (Продукт 6)     Каждая транзакция логируется в ICP Audit Log (часть Compliance). Wallet emits signed transaction records для EU AI Act reporting.                               Wallet → Compliance (tx records)

  Intelligence Layer (Продукт 7)   Wallet emits anonymized transaction events: volume, frequency, asset type. Intelligence использует для fraud model и market analytics.                          Wallet → Intelligence (anonymized events)

**12. Нефункциональные требования**

  ---------------------------- -------------------------------------------------------------------------------------------------------------- -------------------------------------------------------------------------------------------------
  **Категория**                **Требование**                                                                                                 **Как достигается**

  Безопасность ключей          Приватный ключ никогда не существует целиком в одном месте. Компрометация \< t узлов ICP не раскрывает ключ.   ICP threshold ECDSA: 13+ узлов, t-of-n схема. Ключ невосстановим без кворума.

  Производительность подписи   Threshold ECDSA подпись: p50 \< 3 сек, p99 \< 8 сек. Bitcoin broadcast: зависит от mempool.                    ICP subnet latency. Pre-signed transaction pool для повторяющихся паттернов (v1.2).

  Доступность                  99.5% для Wallet API (Bitcoin confirmations зависят от сети).                                                  ICP subnet replication (13+ узлов). Wallet API за load balancer с health checks.

  Upgrade safety               Canister upgrades не теряют state (балансы, политики, ключи).                                                  ic-stable-structures для persistent canister state. Upgrade тестирование на testnet.

  Некастодиальность            Paxio не может заморозить или изъять средства агента без согласия владельца.                                   Canister без admin key после деплоя. Emergency freeze только через Internet Identity владельца.

  Audit immutability           Журнал транзакций Security Sidecar не может быть изменён или удалён.                                           ICP canister append-only log. Хеш каждой записи в следующей --- tamper-evident chain.
